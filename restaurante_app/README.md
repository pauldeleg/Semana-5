# Sistema basico de gestión de restaurante
# Descripción
es un sistema basico de gestion de un restaurante utilizando Programación Orientada a Objetos en python es el objeto de demostrar el uso de clases, objetos, listas, tipos de datos básicos e importación de módulos dentro de un proyecto organizado por carpetas.

# Estructura del proyecto 
Restaurante_app
modelos
  __init__.py
  producto.py
  cliente.py
servicios
  __init__.py
  restaurante.py
main.py
README.md

# Descripción de los archivos
main.py: es el punto de inicio del programa, aqui se crean los objetos, se agregan y se muestran informacion del restaurante.
modelos/producto.py: representa los productos del restaurante.
modelos/cliente.py: representa a los clientes registrados.
servicios/restaurante.py: encargada de administrar las listas de productos y clientes mediante distintos métodos.
__init__.py: indica que las carpetas modelos y servicios son paquetes de python y permite importar sus modulos.
README.md: describe el proyecto y su funcionamiento.
# Tipo de datos utilizados 
str: para almacenar nombres de productos, clientes y del restaurante.
int: para almacenar la edad del cliente.
float: representa el precio de los productos.
bool: indica si un producto esta disponible.

# Funcionalidad 
Crear productos.
Crear clientes.
Registrar productos en el restaurante.
Registar clientes en el restaurante.
Mostrar la informacion organizada en la consola.
# Reflexión
utilizar identificadores descrptivos facilita la comprension y el mantenimiento del codigo, ya que permite reconocer facilmente la función de cada variable, metodo o clase. Emplear datos adecuados ayuda a representar correctamente la información y evita errores y finalmente dividir el proyecto en módulos mejora la organización del código facilitando la reutilización y de ampliar y mantener.
